# Работа с Claude Fable 5 (после Opus 4.7)

Памятка по переходу с Opus 4.7 на Fable 5 в Claude Code.

## Что такое Fable 5

Первая модель семейства Claude 5, класс Mythos-tier — выше Opus по возможностям.
Заточена под длинные агентные сессии: расследования, отладка, архитектура,
многошаговая работа с субагентами.

Ключевые отличия от Opus 4.7/4.8:

| Аспект | Opus 4.7/4.8 | Fable 5 |
|--------|--------------|---------|
| Thinking | опционален, можно отключить | всегда включён, отключить нельзя |
| Цена | $5/$25 за MTok (4.8) | $10/$50 за MTok — в 2 раза дороже |
| Fast mode (`/fast`) | есть | **нет** |
| Классификаторы безопасности | нет | есть (возможны отказы на чувствительных темах) |

При срабатывании классификатора (кибербез, биология) Claude Code автоматически
откатывается на Opus 4.8 и продолжает сессию на нём. Вернуться: `/model fable`.

## Как включить

```bash
# В сессии
/model fable

# При запуске
claude --model fable
```

Постоянно — в `~/.claude/settings.json` (не синхронизируется этим репо,
настраивается на каждой машине отдельно):

```json
{ "model": "fable" }
```

Алиас `fable` предпочтительнее полного ID `claude-fable-5` — автоматически
указывает на актуальную версию.

## Изменения в повседневной работе

1. **`/fast` не работает** — fast mode остался только на Opus 4.8/4.7.
   Нужна скорость → снижать усилия: `/effort medium`.
2. **Effort по умолчанию `high` — его хватает.** `xhigh` — только для
   действительно сложных задач (на 4.7 многие держали `xhigh` постоянно,
   на Fable это лишний расход).
3. **Субагенты**: в frontmatter агентов можно указать `model: fable` для
   сложных ролей (архитектура, ревью), рутину оставлять на `sonnet`/`haiku`.
   Стратегия из скилла `/subagent-driven-development` («наименее мощная
   модель, которая справится») работает без изменений.
4. **Бюджет**: из-за двойной цены Fable — для оркестрации и сложных решений;
   реализацию по готовой спеке и тесты отдавать субагентам на дешёвых моделях.

## Чек-лист миграции

- [ ] `~/.claude/settings.json`: `"model": "fable"` (на каждой машине)
- [ ] Убрать привычку `/fast` → заменить на `/effort medium` где нужна скорость
- [ ] Пересмотреть дефолтный effort: `high` вместо `xhigh`
- [ ] Проверить хуки/statusline, если они матчат имя модели → добавить `claude-fable-5`
- [ ] В API-коде (если есть): убрать `thinking: {type: "disabled"}` — Fable вернёт 400
