## Оценка токенов

- Проза: `слова × 1.3`
- Код/таблицы: `символы / 4`
- MCP-тул: ~500 токенов схемы

## Классификация

| Корзина | Критерий | Действие |
|---------|----------|----------|
| **Нужно всегда** | Упомянуто в CLAUDE.md, стоит за активным триггером, под текущий стек | Оставить |

| **Редко** | Нет триггера, пересекается с другим, не подходит проекту | Убрать / выгрузить |

## Проблемные паттерны

- **Раздутые description скиллов** — >30 слов, грузятся в контекст всегда.
- **Дубли** — скилл дублирует другой скилл; правило дублирует CLAUDE.md.
- **MCP-перегруз** — самый большой рычаг: 30-тульный сервер весит больше всех скиллов вместе. Особенно Figma, если её тулы реально не нужны в большинстве сессий.
- **Распухание CLAUDE.md** — многословные объяснения, протухшие секции, инструкции, которым место в правилах.

В режиме `--verbose` добавь: токены по каждому файлу, построчную разбивку самых тяжёлых, конкретные дублирующиеся строки между пересекающимися компонентами, список MCP-тулов с оценкой размера каждого.

## Best practices

- **MCP — главный рычаг.** Каждая схема тула ~500 токенов; один многотульный сервер дороже всех скиллов. Если Figma нужна в 1 сессии из 20 — кандидат на отключение по умолчанию.
- **Description'ы грузятся всегда**, даже если скилл ни разу не вызван. Тело — нет.
- **Запускай после изменений** — добавил скилл/правило/MCP → прогони, чтобы поймать распухание рано.
- `--verbose` — для отладки «что именно весит», не для регулярного аудита.
