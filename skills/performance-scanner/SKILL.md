## Правила
- **НИКОГДА не редактируй файлы** — только анализ и рекомендации
- Каждую проблему подкрепляй файлом и номером строки
- Оценивай impact: критичный / высокий / средний / низкий
- Не предлагай преждевременную оптимизацию — только реальные проблемы

## Шаг 1: Определение стека и контекста

- Определи язык/рантайм из конфигов
- Оцени масштаб: сколько пользователей, данных, запросов
- Проверь наличие кэширования, очередей, воркеров

## Шаг 2: Сканирование по категориям

### Node.js проблемы
Ищи:
- **Блокирующие операции**: `fs.readFileSync`, `execSync`, тяжёлые sync-функции в event loop
- **Await в цикле**: `for...await` вместо `Promise.all` — N последовательных запросов вместо параллельных
- **Memory leaks**: растущие массивы/объекты без очистки, глобальные переменные-кэши без лимитов, listeners без removeListener
- **Тяжёлые зависимости**: moment.js (→ dayjs), lodash целиком (→ lodash-es или нативные методы)
- **Неэффективные регулярки**: ReDoS паттерны, regex в горячих циклах
- **Отсутствие стриминга**: загрузка больших файлов в память целиком вместо stream
- **N+1 запросы**: цикл с запросом к БД/API внутри
- **Отсутствие connection pooling**: новое соединение на каждый запрос

### SQL / БД проблемы
- `SELECT *` — выборка всех колонок
- Отсутствие индексов на WHERE/JOIN колонках
- N+1 — запрос в цикле
- Нет LIMIT на потенциально большие выборки
- Тяжёлые JOIN без необходимости

### PHP проблемы
- `file_get_contents` для HTTP запросов (→ curl с таймаутами)
- Отсутствие opcode cache
- `include`/`require` в циклах
- Неоптимальные циклы по большим массивам
- Отсутствие кэширования результатов БД

### Go проблемы
- Горутины без лимитов (goroutine leak)
- Аллокации в горячих путях
- Неиспользование sync.Pool
- Конкурентный доступ без мьютексов

### Общие проблемы
- Отсутствие таймаутов на внешние HTTP запросы
- Повторные вычисления одного и того же
- Логирование в горячих путях (I/O)
- Большие payload без пагинации
- Отсутствие gzip/compression

## Шаг 3: Оценка impact

Для каждой найденной проблемы оцени:
- **Частота**: как часто этот код выполняется
- **Объём данных**: сколько данных обрабатывается
- **Блокирование**: блокирует ли другие операции
- **Impact**: критичный (падение/зависание) / высокий (заметная задержка) / средний (неоптимально) / низкий (можно улучшить)
