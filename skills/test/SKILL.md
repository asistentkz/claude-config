Напиши тесты для: $ARGUMENTS

## Правила
- **Сначала покажи план тестов** — какие файлы создашь, что покроешь
- **Жди одобрения** перед созданием файлов
- Не трогай код проекта — только создавай тестовые файлы

## Порядок работы

### 1. Анализ (read-only)
- Прочитай тестируемый код
- Определи публичные функции / эндпоинты
- Найди зависимости (БД, API, файлы)
- Определи edge cases и граничные условия

### 2. Определение стека тестирования
Определи по проекту:
- **Node.js** → Jest / Mocha / Vitest
- **PHP** → PHPUnit
- **Go** → go test + testify
- **Flutter/Dart** → flutter test
- **Python** → pytest
- Если не определяется — спроси у пользователя

### 3. План тестов (показать пользователю)
```
Модуль: [название]
Тестовый фреймворк: [определённый по стеку]
Файлы для создания:
  - tests/[module].test.[ext]

Тест-кейсы:
  ✅ [функция] — нормальный сценарий
  ✅ [функция] — пустые данные
  ✅ [функция] — невалидный ввод
  ❌ [функция] — не тестируем (причина)

Зависимости: нужен mock для БД / API / ...
```

**СТОП — жду одобрения перед написанием**

### 4. Написание тестов (после одобрения)

Каждый тест следует паттерну AAA:
```
// Arrange — подготовка данных
// Act — вызов тестируемого кода
// Assert — проверка результата
```

Правила написания:
- Минимальные моки — тестировать реальную логику
- Фокус на критичных путях
- Понятные имена тестов — описывают что проверяется
- Каждый тест проверяет одну вещь
