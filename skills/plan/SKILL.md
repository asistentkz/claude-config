Составь план по задаче и проверь его через Fable: $ARGUMENTS

Три шага, не пропускай:

## Шаг 1 — Сбор информации (Sonnet)
Запусти субагента `researcher` (subagent_type: researcher) с задачей: собрать всё релевантное по «$ARGUMENTS» — какие файлы задействованы, как оно сейчас работает, зависимости, ограничения. Получи карту фактов.

## Шаг 2 — Черновик плана (Opus, главный цикл)
На основе фактов из шага 1 составь план сам (ты на Opus): цель, шаги, файлы к изменению, риски, порядок выполнения, план отката. Покажи черновик пользователю.

## Шаг 3 — Верификация Fable
Отдай черновик на проверку: запусти `subagent_type: architect` (Fable) с полным текстом плана и заданием: «найди дыры, неверные допущения, пропущенные шаги и риски; верни вердикт GO / GO_WITH_CHANGES / REWORK + правки по приоритету».
- Если Fable недоступна / лимит / refusal — повтори тот же вызов с `model: opus` и сообщи, что проверку сделал Opus, а не Fable.

Покажи пользователю вердикт и правки, интегрируй их в финальный план.

**Не начинай реализацию** — `/plan` только планирует и проверяет. Реализация — отдельно (`/build` или вручную по плану).
