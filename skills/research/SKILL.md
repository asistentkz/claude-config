Проведи исследование по теме: $ARGUMENTS

## Правила
- **НИКОГДА не редактируй файлы** — только читай и анализируй
- Все находки подкрепляй конкретными путями к файлам и номерами строк
- Если нашёл критическую проблему (уязвимость, утечка данных) — сообщи сразу

## Порядок исследования

### 1. Поиск релевантных файлов
- Найди все файлы, связанные с темой (Glob, Grep)
- Определи entry points и ключевые модули

### 2. Анализ кода
- Прочитай найденные файлы
- Проследи цепочки вызовов (кто вызывает, кого вызывает)
- Определи зависимости (внешние библиотеки, API, БД)

### 3. Поиск в интернете
- Найди лучшие практики и решения по теме (WebSearch)
- Документация используемых библиотек и API
- Известные уязвимости (CVE) для найденных зависимостей
- Примеры реализации и рекомендации из сообщества
- Сравни подход в коде с современными стандартами

### 4. Оценка
- Качество кода (дублирование, сложность, мёртвый код)
- Безопасность (инъекции, XSS, hardcoded credentials)
- Архитектура (разделение ответственности, связность модулей)
- Сравнение с лучшими практиками из интернета

### 5. Отчёт
Выдай структурированный отчёт:
- **Что нашёл**: краткое описание модуля/фичи
- **Файлы**: список ключевых файлов с описанием
- **Проблемы**: найденные проблемы с приоритетом (критично / высокий / средний / низкий)
- **Зависимости**: внешние API, библиотеки, таблицы БД
- **Рекомендации**: что нужно сделать
- **Из интернета**: лучшие практики, решения, полезные ссылки
