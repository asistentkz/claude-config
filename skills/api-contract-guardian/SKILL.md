## Правила
- **НЕ редактируй файлы без одобрения** — сначала покажи что нашёл и план исправлений
- Проверяй и входящие (от клиентов) и исходящие (к внешним API) данные
- Конкретные файлы и строки для каждой проблемы

## Шаг 1: Обнаружение API интеграций

Найди все точки интеграции:

### Исходящие (проект → внешний API)
Grep паттерны:
- `fetch(`, `axios`, `got(`, `request(`, `http.get`, `https.get` — HTTP клиенты
- `file_get_contents('http`, `curl_init` — PHP
- `http.Get`, `http.Post` — Go
- Конкретные API: Trello, Google Sheets, OpenAI, AmoCRM, Cloudinary, Telegram

### Входящие (клиенты → проект)
- Express роуты: `app.get`, `app.post`, `router.get`, `router.post`
- Webhook эндпоинты: `/webhook`, `/callback`
- Telegram handlers: `bot.on`, `bot.onText`, `bot.command`

### Межсервисные
- Вызовы между ботами/сервисами внутри экосистемы

## Шаг 2: Аудит валидации

Для каждой найденной интеграции проверь:

### Исходящие запросы
- [ ] Есть ли проверка статуса ответа (response.ok, status code)
- [ ] Есть ли try/catch или .catch() на запрос
- [ ] Есть ли таймаут
- [ ] Валидируется ли структура ответа (или слепое доверие `response.data.result`)
- [ ] Обработка ошибок API (rate limit, 5xx, невалидный JSON)
- [ ] Retry логика при временных сбоях

### Входящие запросы
- [ ] Валидация тела запроса (типы, обязательные поля)
- [ ] Валидация query/params
- [ ] Санитизация входных данных
- [ ] Проверка авторизации/аутентификации
- [ ] Ограничение размера payload

### Telegram handlers
- [ ] Проверка message/callback_query на наличие ожидаемых полей
- [ ] Обработка случаев когда message.text undefined
- [ ] Обработка callback_data формата
