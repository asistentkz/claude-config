Сгенерируй итоговый отчёт по: $ARGUMENTS

## Правила
- **Не редактируй код проекта** — только создавай файлы отчётов
- Отчёт сохраняй в папку `_audit/` в корне проекта
- Формат: markdown файл с датой в имени

## Порядок работы

### 1. Сбор данных
- Прочитай результаты предыдущих исследований (если есть в `_audit/`)
- Если данных нет — проведи быстрый анализ модуля по всем направлениям:
  - Структура и архитектура
  - Безопасность
  - Запросы к БД
  - Качество кода

### 2. Структура отчёта

```markdown
# Отчёт: [название модуля]
Дата: YYYY-MM-DD

## Обзор
- Назначение модуля
- Количество файлов / строк кода
- Основные зависимости

## Критические проблемы 🔴
[Требуют немедленного исправления]

## Высокий приоритет 🟠
[Блокеры и серьёзные проблемы]

## Средний приоритет 🟡
[Производительность и качество кода]

## Низкий приоритет 🟢
[Техдолг, рефакторинг]

## Безопасность
- Найденные уязвимости с приоритетами

## БД и запросы
- Проблемные запросы
- Рекомендации по индексам

## Оценка трудозатрат
| Направление | Сложность | Примерный объём |
|-------------|-----------|-----------------|

## Рекомендуемый порядок действий
1. ...
2. ...
3. ...
```

### 3. Сохранение
- Сохрани отчёт в `_audit/report_[module]_YYYY-MM-DD.md`
- Если `_audit/` не существует — создай
