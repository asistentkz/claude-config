## Правила
- **НЕ редактируй файлы без одобрения** — сначала покажи план изменений
- Не ломай текущее поведение — новая система должна быть обратно совместима
- Адаптируй под стек проекта (Node.js, Go, PHP, Python)

## Шаг 1: Аудит текущей обработки ошибок

Найди и проанализируй:
- `try/catch` блоки — что ловят, что делают с ошибкой
- `process.on('uncaughtException')`, `process.on('unhandledRejection')` — есть ли глобальные обработчики
- `console.log/error` vs структурированное логирование
- Молчаливые провалы — пустые catch блоки, проглоченные ошибки
- Express middleware для ошибок — есть ли `app.use((err, req, res, next))`
- Telegram bot error handlers — `bot.on('polling_error')`, `bot.catch()`

Grep паттерны для поиска:
- `catch` — все catch блоки
- `console.error`, `console.log.*error` — текущее логирование
- `process.on` — глобальные обработчики
- `throw` — что и где бросается
- Пустые catch: `catch.*\{\s*\}` или `catch.*{}`

## Шаг 3: Предложение стандарта

Предложи единую систему, адаптированную под стек:

### Для Node.js:
1. **Класс ошибок** — базовый AppError с кодом, сообщением, контекстом
2. **Логгер** — winston/pino вместо console.log, уровни: error/warn/info/debug
3. **Глобальный обработчик** — uncaughtException, unhandledRejection с логированием и graceful shutdown
4. **Express middleware** — централизованный обработчик ошибок
5. **Telegram bot handler** — единый обработчик ошибок бота
6. **Формат лога** — timestamp, level, message, context, stack

### Для Go:
1. Пакет errors с кастомными типами
2. Structured logging (slog/zerolog)
3. Middleware для recovery

### Для PHP:
1. set_error_handler + set_exception_handler
2. PSR-3 логгер
3. Единые коды ошибок

## Шаг 4: План внедрения

Покажи конкретный план:
```
1. Создать: [путь] — [описание утилиты/middleware]
2. Изменить: [путь] — [что поменять]
...
```

**Приоритет внедрения:**
1. Глобальные обработчики (быстрая победа — ловит всё что падает)
2. Логгер (замена console.log)
3. Пустые catch блоки (убрать молчаливые провалы)
4. Классы ошибок (стандартизация)

Дождись одобрения перед внедрением.
