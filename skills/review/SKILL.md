Проведи ревью перед запуском: $ARGUMENTS

## Правила
- **НИКОГДА не редактируй файлы** — только читай и анализируй
- Если нашёл блокер — сразу скажи **НЕ ЗАПУСКАТЬ** с объяснением
- Вердикт в конце: ✅ можно запускать / ⚠️ с оговорками / 🛑 нельзя запускать

## Что проверять

### 1. Ошибки и баги
- Синтаксические ошибки
- Undefined переменные, несуществующие функции
- Неправильные типы данных, сравнения
- Незакрытые соединения, утечки ресурсов
- Бесконечные циклы, рекурсия без выхода

### 2. Безопасность (быстрая проверка)
- Пользовательский ввод попадает в SQL / HTML / shell без фильтрации?
- Новые credentials в коде?
- Открытые эндпоинты без авторизации?

### 3. Совместимость
- Работает ли на текущей версии рантайма (Node.js / PHP / Go / и т.д.)?
- Нет ли зависимостей от пакетов, которых нет на сервере?
- Конфликты с существующим кодом

### 4. Побочные эффекты
- Затрагивает ли другие модули?
- Меняет ли структуру БД?
- Меняет ли конфиги, права, переменные окружения?
- Может ли сломать существующий функционал?

### 5. Качество
- Логирование ошибок (не молчаливый провал)
- Обработка edge cases (пустые данные, null, большие объёмы)
- Нет ли debug-кода (console.log, var_dump, print, fmt.Println для отладки)

## Формат отчёта

### Вердикт: ✅ / ⚠️ / 🛑

### 🛑 Блокеры (нельзя запускать пока не исправлено)
| # | Проблема | Файл:строка | Почему блокер |

### ⚠️ Предупреждения (можно запускать, но желательно исправить)
| # | Проблема | Файл:строка | Риск |

### ℹ️ Замечания
| # | Замечание | Файл:строка |

### Затронутые модули
- Список модулей/файлов, которые могут быть затронуты изменениями

### Чек-лист перед запуском
- [ ] Бэкап БД сделан
- [ ] Бэкап файлов сделан
- [ ] Протестировано на dev/staging
- [ ] ...
